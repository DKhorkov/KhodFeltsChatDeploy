# Nginx + SSL (HTTPS)

Приложение доступно через nginx reverse proxy с автоматическим SSL-сертификатом Let's Encrypt.

- **HTTP** (`http://kfc.webtm.ru`) — автоматический редирект на HTTPS
- **HTTPS** (`https://kfc.webtm.ru`) — основной доступ к приложению
- Сертификат обновляется автоматически контейнером `certbot`

## Архитектура

```
Браузер --> nginx (порт 443, SSL) --> kfc (порт 8080, внутри docker-сети)
```

Приложение `kfc` больше не доступно напрямую снаружи — весь трафик идёт через nginx.

## Первоначальная настройка

### 1. Убедиться, что порты 80 и 443 открыты

В панели Timeweb Cloud (или файрволе сервера) должны быть открыты порты **80** и **443** для входящих TCP-соединений.

### 2. Остановить текущее приложение

```bash
task -d scripts down -v
```

### 3. Получить SSL-сертификат

Запустить один раз, указав email для уведомлений от Let's Encrypt:

```bash
task -d scripts init_ssl SSL_EMAIL=your@email.com -v
```

Таска:
1. Создаёт директории `certbot/conf` и `certbot/www`
2. Поднимает временный nginx для прохождения ACME-challenge
3. Запрашивает сертификат Let's Encrypt для `kfc.webtm.ru`
4. Останавливает временный nginx

### 4. Запустить приложение

```bash
task -d scripts up -v
```

### 5. Проверить

Открыть в браузере: `https://kfc.webtm.ru`

## Обновление сертификата

Сертификат обновляется **автоматически** — контейнер `certbot` проверяет необходимость обновления каждые 12 часов.

Для ручного обновления:

```bash
task -d scripts renew_ssl -v
```

## Команды

| Команда | Описание |
|---------|----------|
| `task -d scripts init_ssl SSL_EMAIL=... -v` | Первоначальное получение SSL-сертификата (один раз) |
| `task -d scripts renew_ssl -v` | Ручное обновление сертификата |

## Файлы

| Файл | Описание |
|------|----------|
| `nginx.conf` | Конфигурация nginx (HTTP-редирект, SSL, проксирование, WebSocket) |
| `certbot/conf/` | Сертификаты Let's Encrypt (создаётся автоматически) |
| `certbot/www/` | Директория для ACME-challenge (создаётся автоматически) |

## Смена домена

Если домен изменится:

1. Обновить `server_name` в `nginx.conf` (2 места)
2. Обновить домен в таске `init_ssl` в `scripts/Taskfile.yml`
3. Удалить старые сертификаты: `rm -rf certbot/`
4. Повторить шаги из раздела "Первоначальная настройка"

## Устранение проблем

**nginx не запускается с ошибкой о сертификате:**
Сертификат ещё не получен. Запустите `task -d scripts init_ssl SSL_EMAIL=... -v`.

**Ошибка "too many certificates" от Let's Encrypt:**
Let's Encrypt имеет лимит 5 сертификатов на домен в неделю. Подождите или используйте `--staging` флаг для тестирования.

**WebSocket не работает:**
Проверьте, что в `nginx.conf` есть заголовки `Upgrade` и `Connection`. Они уже настроены в текущей конфигурации.
